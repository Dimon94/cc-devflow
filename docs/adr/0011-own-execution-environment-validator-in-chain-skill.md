# Own Execution Environment Validator In Chain Skill

cc-devflow will own the Execution Environment Validator in the `execution-environment-contract` Chain Skill instead of `cc-dev`, `cc-plan`, or `lib/compiler`. The validator checks only the Execution Environments portion of `task.md`: required sections, environment rows, child dispatch boundaries, assigned task coverage, dependency readiness, and running sibling touch or mutable-resource conflicts. It emits a read-only fact report with `error`, `blocker`, and `warning` entries plus stable short codes; it does not parse the whole `task.md`, choose dispatch strategy, read Git state, create threads, route workflow stages, or make quality verdicts. `cc-plan` consumes the report before freeze and `cc-dev` consumes it before dispatch; `cc-do` receives already-scoped work and must not duplicate this gate.
